# Palworld Breeding Calculator — Visual Style Rationale

> 阶段：06-design  
> 项目：palbreedingcalculator.net  
> 目标市场：US / Global English，Palworld 玩家社区  
> ICP：效率导向型玩家（硬核 min-max + 休闲快速查询）

---

## 1. 风格方向对比

### 方向 A：官方游戏风（Fantasy RPG UI）
- **视觉特征**：厚重边框、石质纹理、羊皮纸色、金属高光、兽爪/爪印图标。
- **优点**：玩家代入感强，一眼识别是 Palworld 相关。
- **缺点**：
  - 极易触碰版权/商标红线（官方 UI 语言、元素形状）。
  - 信息密度低，加载重，不适合移动端快速查询。
  - 与竞品中的 Wiki 风趋同，难以差异化。
- **结论**：❌ 弃用。PRD 明确 NOT-DO 不使用官方素材；且工具型站点需要快速而非沉浸。

### 方向 B：极简 SaaS 风（Default AI Look）
- **视觉特征**：Inter 字体、白底、紫色/蓝色渐变、圆角卡片、居中 Hero + 三列特性、大量留白。
- **优点**：干净、现代、开发成本低。
- **缺点**：
  - 毫无游戏社区氛围，玩家不会觉得“这是给我用的工具”。
  - 与无数 AI SaaS 撞脸，信任感弱。
  - 游戏玩家预期看到“数据密度”和“可操作性”，而不是“营销页”。
- **结论**：❌ 弃用。不符合站点类型（工具 + 数据密度）和 ICP。

### 方向 C：战术终端风（Tactical Game Toolkit）
- **视觉特征**：
  - 深色背景（#0B0F12）+ 高对比文字
  - 军火/终端感的 UI：清晰边框、标签化信息、紧凑数据卡片
  - 类型色编码（Fire=红、Water=蓝、Grass=绿、Electric=黄、Ice=青、Ground=棕、Dark=紫、Dragon=靛、Neutral=灰）
  - 几何抽象头像/首字母色块代替官方 Pal 素材
  - 等宽数字字体用于 CombiRank 等数据
- **优点**：
  - 符合“硬核玩家”对效率工具的预期（像游戏内终端/建造菜单）。
  - 信息密度高，首屏即算，移动端友好。
  - 与竞品普遍偏 Wiki/营销风形成差异。
  - 完全规避官方素材，版权安全。
- **缺点**：需要精细控制对比度和色盲友好性。
- **结论**：✅ 采用。

## 2. 设计原则

1. **工具优先，不是营销优先**  
   首屏直接给计算入口，不先放长文案。Hero 只保留一句话价值 + 两个方向入口。

2. **数据可读性 > 装饰**  
   CombiRank、类型、Work Suitability、被动技能等数据必须一眼可扫描。用表格/标签/色块，不用大段文字。

3. **移动端即主屏**  
   按钮 44px 以上，输入框大，选择器触屏友好。结果卡片在手机上单列堆叠。

4. **类型色作为系统语言**  
   每个 Pal 元素类型都有固定色，颜色帮助快速识别，而不是装饰。

5. **无官方素材**  
   全部使用抽象几何头像、首字母色块、SVG 类型图标。避免任何官方 Pal 形象。

## 4. 关键设计决策

| 决策 | 选择 | 理由 |
|---|---|---|
| 主背景 | 深色 #0B0F12 | 游戏工具感，减少眼疲劳，适合夜间边玩边查 |
| 主强调色 | 琥珀 #F5A623 | 游戏中常见“稀有/高价值”提示色，CTA 醒目 |
| 次要强调 | 青绿 #4ECDC4 | 成功、数据 freshness、版本徽章 |
| 表面层级 | 三层：page / card / elevated | 工具页卡片多，需清晰分层 |
| 字体 | Inter + JetBrains Mono | Inter 用于 UI，JetBrains Mono 用于 CombiRank/数值 |
| 圆角 | 8px 为主，12px 大卡片 | 不过度圆润，保持工具利落感 |
| 阴影 | 仅 elevated 元素，低调 | 深色模式阴影不宜过重 |
| 头像 | 抽象几何 SVG 色块 + 首字母 | 规避版权，统一生成 |

---

## 5. 与竞品差异

- Palworld Wiki：信息密集但像数据库，没有现代 UI。
- Palworld.gg：极简但缺乏个性。
- PinDrop / Palpedia：商业化重，广告感强。
- **本站点**：像玩家自己的战术工具终端，快、可分享、无干扰。

---

## 6. 待确认项

- 是否接受深色模式为默认？（可后续加入 light toggle，但 v0 默认 dark）
- 类型色是否需要为色盲用户增加图案/纹理区分？
- 是否需要自定义 Pal 抽象头像生成规则？

**结论：采用战术终端风，v0 默认深色，类型色编码，几何抽象头像，工具优先布局。**
